## 一、Server    
  
1、垃圾回收改进:  
  
create index 不管多久都不会影响垃圾回收了. 以前的版本有个函数getoldestxmin获取实例中最老的未结束事务号, 这个事务号以后产生的垃圾不能被回收, 所以create index如果花了1小时, 这1小时内实例所产生的垃圾要等index创建结束后才能回收.   
  
当一个表里面有dead tuple的PAGEs占比低于BYPASS_THRESHOLD_PAGES时, 将跳过此次垃圾回收对这个表的index的vacuum. 因为就算访问到了这些dead tuple对应的索引, 回表时也不会额外的访问很多heap page.  
  
2、角色  
  
Add "pg_database_owner" default role. 表示数据库owner  
  
新增角色 pg_read_all_data (MySQL 兼容性之 - 读写用户的只读影子用户 - 默认读权限  ) , pg_write_all_data (写权限)  
  
3、连接  
  
client_connection_check_interval , 协议层支持心跳包, 如果客户端已离线 可以 快速中断 运行中的长SQL - Detect POLLHUP/POLLRDHUP while running queries  
  
4、crash清理  
  
自动清理临时文件 after backend crash restart - GUC remove_temp_files_after_crash  
     
### 1、分区表    
    
1、分区表DML, DQL性能大幅度增强  
  
分区表性能增强, Avoid creating duplicate cached plans for inherited FK constraints. 节约cache  
分区表性能再次增强 - ExecInitModifyTable 分区裁剪精细化     
优化器增强 Rework planning and execution of UPDATE and DELETE. (减少传导不必要的列value、避免为每个分区生成subplan)  
    
2、分区表DDL时支持2PC处理, 减少锁冲突  
  
分区表attach, detach分区支持无锁(短暂锁) - 基于2阶段routines管理  
  
### 2、Indexes    
    
1、索引在高频更新场景中的性能大幅提升  
  
频繁更新索引列引起的索引分裂和膨胀问题大幅度缓解, Enhance nbtree index tuple deletion - bottom-up index deletion    
  
2、GiST SP-GiST 索引创建速度大幅提升  
  
gist和sp-gist索引AM支持sort接口, 大幅加速GiST和SP-GiST 索引build速度  
  
3、SP-GiST 索引支持include功能, 支持任意维度聚集  
  
SP-GiST 索引新增 index 叶子结点 include column value 功能 支持  
  
4、BRIN 索引支持多区间, 支持布隆过滤器. 性能和适应场景增强  
  
BRIN (典型IoT 时序场景) 块级索引支持 bloom filter - 随机,大量distinct value, 等值查询  
BRIN (典型IoT 时序场景) 块级索引支持 multi-range min-max  - 分段范围索引  
    
### 3、优化器    
    
增强or, and多条件选择性评估 - 使用multiple extended statistics  
Extended statistics on expressions 优化器增强: 自定义扩展统计信息支持 指定 表达式 - 按 表达式 收集统计信息组合    
  
    
### 4、通用性能提升    
高并发场景性能大幅提升, 主要解决getsnapshotdata在上万连接后的性能急剧下降问题. 5万连接下性能无损. - GetSnapshotData 高并发优化, 以及 64位xid避免xid wrap优化  
  
seqscan io chunk优化, 避免离散IO  
  
老外惊呼: 世界变化快! PostgreSQL 14 on ARM 8.1 加上LSE patch性能飙升到140万tps  
  
支持 vacuum 强制加速参数 vacuum_failsafe_age , vacuum_multixact_failsafe_age.  防止 xid wraparound  
  
TOAST 支持 lz4 压缩算法 - --with-lz4 , 新增GUC default_toast_compression  
  
大表search IN ( consts ) - linear search TO hash table probe (consts 个数>= MIN_ARRAY_SIZE_FOR_HASHED_SAOP)  
    
### 5、监控    
1、支持memory context内容监控  
  
查看backend process的memory context  
打印其他会话的memory context, 诊断内存消耗问题 - pg_backend_memory_contexts(pid)
